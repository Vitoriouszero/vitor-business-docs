REGRAS CRÍTICAS DE PROMPT — Veo 3.1 + imagen_edit (validadas empiricamente, Criativos 01–10)
Rule 1: não usar second_image_path para corrigir defeitos — fossiliza o problema.
Rule 2: proibições específicas > genéricas ("do NOT add elongated nozzle").
Rule 3: fórmula Manus: [Composição] + [Produto] + [Cena/Pessoa].
Rule 4: generate_with_reference é o modo mais poderoso para UGC com pessoa.
Rule 5: swap_background dá máxima fidelidade geométrica para shots sem pessoa.
Rule 6: remove_background retorna fundo branco, não transparente.
Rule 7 (DEPRECATED): instrução negativa de mão livre ignorada pelo Veo → ver Rule 9.
Rule 8: direção de bocal/ponta deve ser explicitada para produtos funcionais.
Rule 9: membro livre → ação positiva com timestamp ("left hand resting on her hip").
Rule 10: shot type front-loaded como primeiras palavras do prompt.
Rule 11: timestamp prompting obrigatório para controle de ação, emoção e membros.

REGRA CRÍTICA: veo_get_status e creative_approve exigem o FULL operation path. Short ID (UUID curto) retorna 404. Copiar verbatim o campo operation_id do retorno.
